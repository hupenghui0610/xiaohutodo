1. **APP_ID**: `cli_a927c34524f8dbef` ✅ 已配置
2. **APP_SECRET**: 通过环境变量配置 ✅ 
3. **TABLE_ID**: `tblZKtePBKg6T21x` ✅ 已配置（多维表格 ID）
4. **DATA_TABLE_ID**: `tblEnUS4h7MN8Yor` ⚠️ 需要替换为你的实际数据表 ID

---

## 🔍 如何获取数据表 ID（DATA_TABLE_ID）

### 方法 1：从 URL 获取（最简单）

1. 打开你的飞书多维表格：https://feishu.cn
2. 找到多维表格 `tblZKtePBKg6T21x`
3. 打开其中的一个数据表
4. 查看浏览器地址栏的 URL，格式类似：
   ```
   https://xxx.feishu.cn/base/tblZKtePBKg6T21x?table=tblXXXXXXXXXXXX&view=vewXXXXXXXXXXXX
   ```
5. `table=` 后面的就是数据表 ID（以 `tbl` 开头）

### 方法 2：使用飞书 API 获取

1. 访问飞书开放平台 API 调试工具
2. 调用接口：`GET /open-apis/bitable/v1/apps/{app_token}/tables`
3. 返回结果中会列出所有数据表及其 ID

## 🗂️ 数据表字段配置

在飞书多维表格中，需要创建以下字段：

| 字段名 | 字段类型 | 说明 |
|--------|----------|------|
| `id` | 文本 | 待办唯一标识 |
| `type` | 文本 | 待办类型（A/B/C） |
| `title` | 文本 | 待办标题 |
| `done` | 复选框 | 是否完成 |
| `date` | 日期 | 日期（C类待办使用） |
| `weekStart` | 日期 | 周起始日期（B类待办使用） |
| `delayed` | 复选框 | 是否延迟（B类待办使用） |
| `createdAt` | 日期时间 | 创建时间 |

### 创建字段步骤：

1. 打开飞书多维表格
2. 点击表格右上角的 "+" 添加字段
3. 按照上表创建所有字段
4. 字段名必须完全一致（区分大小写）

## ❓ 常见问题

### Q: 找不到数据表 ID？
A: 确保你已经在多维表格中创建了至少一个数据表。如果是新建的多维表格，会自动创建一个默认表。

### Q: API 返回 403 错误？
A: 检查飞书应用权限是否正确配置：
- `bitable:app`
- `bitable:app:readonly`
- `bitable:app:readwrite`

### Q: 数据保存失败？
A: 检查：
1. 数据表字段是否全部创建
2. 字段名是否完全一致
3. 字段类型是否正确
